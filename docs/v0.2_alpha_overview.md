# MemoryQwen v0.2-alpha — Adaptive Cognitive Exoskeleton

## What v0.2-alpha is

MemoryQwen v0.2-alpha is the first Adaptive Cognitive Exoskeleton (ACE-v1) checkpoint.

Instead of having a single local model handle every question the same way, ACE-v1 routes each query through a unified cognitive pipeline that decides what context, which model role, and what safety checks to apply.

## What ACE-v1 solves

- **Capability questions** → routed to Capability Registry (authoritative facts, not guessed from old training data)
- **Latest information** → web route detected, web evidence brought in as temporary [W] context
- **Complex planning** → deep_suggested route, --deep flag available for 14B deep mode
- **Hallucination risks** → judge_review route, guard always enabled
- **Source conflicts** → manual_review route, Registry always prioritized

## What ACE-v1 does NOT solve

> ACE-v1 does not claim to solve the full abstract reasoning wall.
> ACE-v1 shifts part of the reasoning burden from a single model to the system layer.
> 14B deep mode helps with complex reasoning but is not a replacement for architectural innovation.

## Model Roles

- **7B (qwen2.5:7b)** — default daily model
- **14B (qwen2.5:14b)** — optional deep mode
- **14B is not required for base usage**

## Architecture

```
User question
  → ACE Controller
    → Capability Registry (facts)
    → TDR-v1 (difficulty routing)
    → Retrieval Gate (smart memory access)
    → Web Evidence (controlled web)
    → Model Role (7B/14B)
    → Guard (safety audit)
    → Review (judge/manual)
  → Answer + full metadata
```

## Current Limitations

- No Web UI
- No PDF/DOCX ingestion
- No embedding/vector DB
- No crawler
- No automatic 14B escalation
- No automatic web access
- No SDGI (Semantic Depth-Graded Inference) yet
