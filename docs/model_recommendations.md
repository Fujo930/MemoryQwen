# 模型推荐

## 推荐等级

| 模型 | 推荐用途 | 通过率 (capability boundary) | 大小 |
|------|---------|---------------------------|------|
| qwen2.5-coder:3b | smoke test / 低资源验证 | ~64% | 1.9GB |
| **qwen2.5:7b** | **默认常驻模型** | **~91%** | 4.7GB |
| qwen2.5:14b | deep mode / reasoning | 待测试 | ~8GB |
| 32B+ | 实验 / 不推荐默认 | — | 10GB+ |

## 选择指南

### 用 3B 的场景
- 快速验证系统是否跑通
- 低 VRAM 设备 (≤6GB)
- CI / smoke test
- 不要求高准确率

### 用 7B 的场景（推荐）
- 日常聊天
- 功能边界判断
- 资料检索问答
- 纠错和策略遵从

### 用 14B 的场景
- 复杂推理
- 需要更深入的分析
- GPU 空闲时

### 不用 32B+ 作为默认
- v0.1 目标 RTX 4080 16GB
- 32B+ 显存要求过高
- 不符合"7B 常驻"设计目标

## 实际测试数据

基于同一套 capability boundary 20 题测试：

| 问题类型 | 3B 准确率 | 7B 准确率 |
|---------|----------|----------|
| PDF / 文件格式 | ❌ 幻觉"支持" | ✅ 正确"不支持" |
| Guardian / daemon | ❌ 混淆 | ✅ 正确区分 |
| embedding / vector | ⚠️ 回避 | ✅ 正确"不支持" |
| crawler / archive | ❌ 混淆 | ✅ 正确区分 |
| 功能边界整体 | 64% | 91% |

结论：**3B 跑通，7B 常驻，14B 深度，32B+ 实验。**
